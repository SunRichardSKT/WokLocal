# 投稿指南

感谢你把真实、能复现的海外厨房经验分享给更多留子。完全不会代码也没关系，菜谱、食材替代、厨具建议和落地经验都可以通过网页表单提交。

## 最简单的投稿方法

1. 打开线上网站的“我要投稿”：`https://SunRichardSKT.github.io/WokLocal/contribute/`。
2. 选择“分享一道菜”“补充一种食材”“推荐一件厨具”或“分享落地经验”。
3. 像填写普通问卷一样填写内容。看不懂或不确定的选填项可以留空，ID 和文件路径会自动生成。
4. 点击“检查填写内容”，页面会把遗漏标在对应区域。
5. 有 GitHub 账号就点击“用 GitHub 提交”；没有账号就点击“使用邮件投稿”。

草稿会自动保存在当前浏览器，刷新页面后可以继续填写。本地选择的图片不会被保存，刷新后需要重新选择。

## 菜谱怎么写

- 菜名：中文或英文至少填写一个。
- 简介：用一句话说明这道菜适合什么场景。
- 食材：优先搜索现有食材库；找不到时选择“未入库食材”，填写中文或英文名和用量。
- 厨具：勾选实际需要的厨具。
- 步骤：一条只写一个主要动作，火候、时间和判断熟没熟尽量具体。
- 图片：封面优先横图，步骤图只放关键动作；可以上传本地图片并在浏览器中压缩。
- 选填内容：拼音、菜系、视频、新手提示、踩坑说明都可以留空。

每道菜保留标准做法；如果你知道当地替代食材，也请写清商品名、超市或货架位置，以及咸度、甜度、口感或用量差异。

## 食材、厨具和落地经验怎么写

- 食材：中文或英文至少一个即可。本地叫法、替代商品、购买地点、品牌、价格、用量和味道差异知道多少写多少。
- 厨具：中文或英文至少一个即可。可以补充是否值得买、在哪里买、价格区间、住房限制和替代工具。
- 落地经验：标题和主要内容必填，正文可以自由使用 Markdown，适合写超市差异、采购路线、预算、电器限制和避坑经验。

## 图片与版权

优先上传本人拍摄、公共领域或已获得明确授权的图片。表单会要求确认使用权；请填写作者或来源。不要提交从商业菜谱、社交平台或搜索结果直接保存、但没有使用许可的图片。

GitHub 投稿时，把表单下载的压缩图拖入 Issue 正文。邮件投稿时，把图片作为附件发送到 `guyanrichard@qq.com`。

## 给熟悉 GitHub 的贡献者

表单的“高级选项”可以复制 YAML、Issue 内容和目标路径，也可以下载 YAML 文件。手动编辑数据时：

- 菜谱放在 `data/recipes/`，通过 `ingredient_id` 和 `equipment.required_ids` 引用共享数据。
- 食材放在 `data/substitutions/`，地区键使用 `uk`、`north_america`、`europe`、`australia` 或 `japan_korea`。
- 厨具放在 `data/equipment/`，落地清单放在 `data/starter-packs/`。

提交前运行：

```bash
npm run validate:data
npm run typecheck
npm run build
```

## 内容原则

- 优先写给零基础新手看，用自己的语言描述做法。
- 购买建议尽量具体到超市、货架区域或线上平台。
- 替代建议必须说明差异，避免只写“可以替代”。
- 不复制商业菜谱原文，不搬运没有许可的图片或视频。
- 视频只提交原始外链，不复制视频内容。
